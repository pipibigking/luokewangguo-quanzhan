# 精灵管理系统 - 产品需求文档（PRD）

## 一、引言

### 1.1 项目背景
本系统是为洛克王国异色精灵交易平台设计的后台管理系统，旨在帮助游戏管理员高效管理精灵数据和公告内容。

### 1.2 项目目标
- 提供直观易用的精灵卡片管理界面
- 支持精灵信息的增删改查及上架/下架操作
- 提供简洁的公告管理功能
- 保持与现有前端风格的一致性

---

## 二、目标用户

| 用户角色 | 核心需求 | 使用场景 |
|---------|---------|---------|
| 游戏管理员 | 管理精灵数据、发布公告 | 日常运营维护、活动发布 |

---

## 三、功能需求

### 3.1 精灵管理模块

**3.1.1 精灵列表**
- 展示所有精灵卡片（包含已下架的）
- 支持按名称、属性、分组搜索/筛选
- 显示精灵基本信息（名称、图片、价格、状态）

**3.1.2 添加精灵**
- 精灵名称输入
- 精灵图片上传
- 精灵属性选择（支持多选）
- 购买价格设置
- 精灵描述编辑
- 能力特点编辑
- 分组选择/自定义

**3.1.3 编辑精灵**
- 修改上述所有字段

**3.1.4 状态管理**
- 上架：设置精灵在用户端可见
- 下架：设置精灵在用户端不可见（后台仍可查看）

**3.1.5 删除精灵**
- 永久删除精灵数据

### 3.2 公告管理模块

**3.2.1 公告编辑**
- 单条公告内容编辑
- 支持纯文本编辑

**3.2.2 公告预览**
- 实时预览公告效果

### 3.3 界面导航

**3.3.1 侧边栏导航**
- 精灵管理入口
- 公告管理入口

---

## 四、非功能需求

### 4.1 性能需求
- 页面响应时间 < 200ms
- 图片加载优化

### 4.2 设计风格
- 现代简约的后台管理风格（参考用户提供的设计图）
- 柔和的渐变色彩
- 圆润的卡片设计
- 清晰的视觉层次

### 4.3 技术架构
- **前端框架**: Vue 3 + TypeScript
- **样式方案**: Tailwind CSS
- **构建工具**: Vite
- **状态管理**: Vue Composition API
